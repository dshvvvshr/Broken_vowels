# Contributing to Broken_vowels

Thank you for your interest in contributing! This project is dedicated to the public good, and we welcome contributions that align with our core directive.

## Core Directive

All contributions must honor the fundamental principle:

> **"No action may interfere with another person's inalienable right to pursue happiness."**

## How to Contribute

### Reporting Issues

1. Check if the issue already exists
2. Use the issue templates provided
3. Provide clear, detailed information
4. Be respectful and constructive

### Submitting Changes

1. **Fork the repository**
2. **Create a feature branch** from `main`
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes**
   - Write clear, maintainable code
   - Follow existing code style and conventions
   - Add tests if applicable
   - Update documentation as needed
4. **Test your changes**
   - Run existing tests: `npm test` (Node.js) or `python -m pytest` (Python)
   - Ensure all tests pass
   - Add new tests for new functionality
5. **Commit your changes**
   - Use clear, descriptive commit messages
   - Reference relevant issues
6. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```
7. **Open a Pull Request**
   - Provide a clear description of the changes
   - Reference any related issues
   - Be responsive to feedback

### Code Style

- **Python**: Follow PEP 8 guidelines
- **JavaScript/Node.js**: Follow existing code style (2-space indentation)
- **Documentation**: Use clear, accessible language

### Testing

- All code changes should include appropriate tests
- Tests should pass before submitting a PR
- For Python: `python -m pytest test_governance.py -v`
- For Node.js: `npm test`

### Documentation

- Update README.md if adding new features
- Keep documentation clear and accessible
- Add examples where helpful
- Update relevant documentation files (EXAMPLES.md, FAQ.md, etc.)

4. **Testing**

   - Real-world implementation examples
